## What worked — first run

`make up` runs the core wiki-first: the **ohmyboring** RAG engine starts; **hermes-agent**
joins automatically if its image exists, otherwise startup falls back to core-only
(so `make ask` works without the agent). pgvector (vector + graph RAG) is opt-in —
`BORING_VECTOR=on make up` brings up Postgres via `--profile vector`.

The host **Ollama** (`ollama serve`) must be up first so the container reaches
embeddings/synthesis via `host.docker.internal`. Models: `gemma4:12b` + `bge-m3`.

## Gotcha

Running bare `docker compose up` expected the external hermes-agent image (Nous
Hermes Agent, not in the repo). Use `make up` instead — it degrades to core-only
when that image is missing rather than hard-failing.
